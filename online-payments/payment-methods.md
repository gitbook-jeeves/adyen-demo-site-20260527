# Payment methods

Payment-method documentation is more useful when it is not isolated from the rest of the integration story. Teams need to understand which methods they plan to support, how those methods affect checkout UX, and which markets or compliance rules come with them.

## Documentation questions this page should answer

* Which methods matter for the target market mix?
* Which methods change the checkout flow or redirect behavior?
* Which methods influence tokenization, recurring use, or authentication?
* What should support and operations teams know once those methods go live?

{% hint style="success" %}
For an AI assistant demo, payment methods are useful because they create natural prospect questions like "which methods need redirect flows?" or "how does recurring behavior vary by method?"
{% endhint %}
