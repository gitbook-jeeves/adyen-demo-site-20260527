# Store rollout

## A rollout model that scales

{% stepper %}
{% step %}
### Pilot a narrow footprint

Start with a small number of stores, one operating model, and explicit success criteria around transaction quality, support burden, and network behavior.
{% endstep %}

{% step %}
### Lock the support path

Define how store teams escalate device issues, how support teams inspect transaction state, and how networking incidents are triaged.
{% endstep %}

{% step %}
### Expand by repeatable unit

Roll out by store archetype, region, or connectivity pattern so the docs stay reusable and operational learnings are easy to capture.
{% endstep %}
{% endstepper %}

## Pages a rollout lead expects to find

- pre-go-live checklist
- network validation checklist
- device assignment and ownership model
- refund and receipt support rules
- post-launch monitoring guide
