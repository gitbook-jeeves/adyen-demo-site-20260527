# Operations and go-live

The live portal does a lot of work after the first integration step. This page makes that visible in the payments entry section.

## What teams need after implementation starts

- Integration and go-live checklists
- Testing tools and sandbox behaviors
- Webhook validation and event tracing
- Refund, capture, and dispute operations
- Cross-functional handoff between engineering, support, finance, and risk

{% hint style="warning" %}
This is one of the areas where payment docs often break down. Technical setup lives in one place, while support and finance workflows get scattered elsewhere. The demo deliberately reconnects them.
{% endhint %}
