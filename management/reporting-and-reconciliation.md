# Reporting and reconciliation

Reporting content works best when it helps a team answer operational questions, not just locate a file or dashboard.

## The questions this section should answer

- Which report or export is the source for this question?
- How does this tie back to transaction events?
- What is delayed, what is final, and what may still change?
- Which team owns investigation when numbers do not match?

{% stepper %}
{% step %}
### Identify the source

Map the question to a report, export, or dashboard view.
{% endstep %}

{% step %}
### Tie it back to the payment lifecycle

Link to the relevant checkout or terminal flow so the reader understands what produced the data.
{% endstep %}

{% step %}
### Define the escalation owner

Make it obvious whether the next stop is support, finance ops, or engineering.
{% endstep %}
{% endstepper %}
